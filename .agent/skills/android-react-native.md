# 📱 Android / React Native Skills

## Stack

- **Framework:** React Native (TypeScript) via Expo SDK
- **Navigation:** Expo Router (file-based, similar to Next.js)
- **Styling:** NativeWind (Tailwind CSS for React Native)
- **UI Components:** React Native Paper
- **Auth:** Firebase Auth (`@react-native-firebase/auth` or Expo's Firebase SDK)
- **State:** React Context + hooks (mirrors web app patterns)
- **API:** Shared REST client pointing to the same Express backend

---

## Android-Specific Knowledge

### Build & Run
```bash
cd mobile
npm install
npx expo start          # start dev server
npx expo run:android    # build & launch on emulator / device
npx expo build:android  # production APK / AAB via EAS
```

### EAS Build (Expo Application Services)
```bash
npm install -g eas-cli
eas login
eas build --platform android --profile preview   # APK for testing
eas build --platform android --profile production # AAB for Play Store
```

### Key Config Files
| File | Purpose |
|---|---|
| `mobile/app.json` | Expo config — package name, permissions, icons |
| `mobile/eas.json` | EAS build profiles (development / preview / production) |
| `mobile/android/` | Native Android project (auto-generated by `expo prebuild`) |

### Permissions (add to `app.json` → `android.permissions`)
```json
"permissions": [
  "CAMERA",
  "READ_EXTERNAL_STORAGE",
  "WRITE_EXTERNAL_STORAGE",
  "RECORD_AUDIO",
  "INTERNET"
]
```

## Project Conventions

### File Structure (`mobile/`)
```
mobile/
├── app/                  # Expo Router pages (file = route)
│   ├── (auth)/           # Auth group — login, register
│   ├── (student)/        # Student tab layout
│   ├── (teacher)/        # Teacher tab layout
│   └── _layout.tsx       # Root layout + auth guard
├── components/           # Shared RN components
├── hooks/                # Custom hooks (mirrors web /hooks)
├── lib/                  # Firebase init, API client, utils
├── assets/               # Images, fonts, icons
└── constants/            # Colors, theme tokens
```

### Styling Rules
- Use **NativeWind** (`className`) for layout and spacing.
- Use **React Native Paper** components (`Button`, `Card`, `TextInput`) for interactive elements.
- Mirror the web Tailwind color tokens so themes stay consistent.

### Navigation Pattern
```tsx
// Expo Router — push a screen
import { router } from 'expo-router';
router.push('/student/ai-tutor');

// Protected route guard in _layout.tsx
const { user } = useAuth();
if (!user) return <Redirect href="/login" />;
```

### API Client
- Reuse the same base URL as the web app (`process.env.EXPO_PUBLIC_API_URL`).
- Attach Firebase ID token to every request:
```ts
const token = await auth.currentUser?.getIdToken();
fetch(`${API_URL}/endpoint`, {
  headers: { Authorization: `Bearer ${token}` }
});
```

## Common Gotchas

- **Metro bundler** doesn't support all Node.js built-ins — use RN-compatible alternatives.
- **Cassandra** is server-side only; mobile talks to the REST/WebSocket API, never directly to Cassandra.
- **Firebase** on Android requires `google-services.json` placed in `mobile/android/app/`.
- **NativeWind v4** requires `babel-plugin-nativewind` and `metro.config.js` setup.
- **Expo Go** doesn't support all native modules — use a **development build** for Firebase and camera features.
